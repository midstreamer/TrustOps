# TrustOps v0.1.0-pilot Release Notes

**Release date:** June 2026  
**Codename:** Commercial Pilot Readiness

## Overview

TrustOps v0.1.0-pilot is the first pilot-ready release for SOCaaS and MDR providers. It delivers case-centered SOC workflow, human-AI trust metrics, client value reporting, and Microsoft Sentinel integration — without replacing your SIEM or SOAR.

## Highlights

### Buyer Demo Package
- 20-minute demo script centered on Apex Energy and CASE-GOLDEN
- Buyer narrative, personas, and data dictionary for sales teams

### Microsoft Sentinel Integration
- `POST /integrations/sentinel/alerts` with native field mapping
- `GET /integrations/sentinel/health` connectivity check
- Sample payload and integration documentation

### Trust Metrics v2
- **Trust Calibration Score** — transparent operational indicator (not AI certification)
- High/low confidence decision analytics
- Human-AI disagreement by severity
- Override reason categories
- QA reversal rate and override accuracy

### Client Value Report v2
- Ten buyer-ready sections including AI oversight and trust metrics summary
- Print / Save as PDF via browser
- Client-safe content (no internal QA, no raw AI prompts)

### Pilot Deployment
- Deployment modes: `local-demo`, `pilot-single-tenant`, `pilot-multi-client`
- Health endpoints: `/health`, `/ready`, `/version`
- Pilot setup CLI and security controls documentation

## Upgrade Notes

- Set `APP_VERSION=0.1.0-pilot` and `DEPLOYMENT_MODE` in `.env`
- Optional: set `SENTINEL_API_KEY` for dedicated Sentinel authentication
- Re-seed database for updated report v2 demo content

## Known Limitations

- Trust Calibration Score is an operational metric, not a statistical AI accuracy measure
- PDF export uses browser print (no server-side PDF generation)
- Seed script does not run on existing databases
- Mock AI mode active when `OPENAI_API_KEY` is unset
