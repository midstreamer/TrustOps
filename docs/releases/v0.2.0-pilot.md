# TrustOps v0.2.0-pilot Release Notes

**Release date:** June 2026  
**Codename:** Client Portal & Integration Hardening

## Overview

TrustOps v0.2.0-pilot extends the commercial pilot with a richer client value portal, reliable report generation, hardened Sentinel/webhook ingestion, and developer workflow improvements. It builds on v0.1.0-pilot without changing core positioning: TrustOps manages SOC case work and proves service value — it does not replace your SIEM or SOAR.

## Highlights

### Client Dashboard — SOC Workflow Funnel
- Visual funnel: Alerts Received → Triage → Investigation → Confirmed Incidents
- Configurable period (7 / 30 / 90 days) with trend indicators
- Aligns client-facing metrics with how MDR providers describe SOC workflow

### Client SOC Assistant
- AI chat sidebar on the Client Dashboard — ask questions while scrolling metrics
- Answers grounded in the client's dashboard data (SLA, cases, funnel, incidents)
- Client-safe: no internal QA notes, raw AI prompts, or cross-client data

### Report Reliability
- `POST /reports/{id}/regenerate` — refresh AI-generated report sections
- Fallback normalization prevents empty executive summaries on publish
- Client Dashboard links to the correct latest published report

### Integration Hardening
- Alert deduplication prevents duplicate cases from replayed webhooks
- Integration event log (`GET /integrations/logs`) for manager troubleshooting
- Sample Azure Logic App workflow for Sentinel forwarding

### Trust Metrics Filters
- Filter by client and date range
- Weekly trend charts for pilot QBR reviews

### Developer Experience
- Separate Next.js cache directories prevent dev/build corruption
- `npm run build:check` validates TypeScript while dev is running

## Upgrade Notes

1. Set `APP_VERSION=0.2.0-pilot` in `.env`
2. Run migrations: `cd backend && alembic upgrade head`
3. Restart backend and frontend
4. For existing deployments with empty published reports, use **Regenerate** on the report detail page (manager role)

```bash
cd backend && alembic upgrade head
# Optional: regenerate Apex report via API or UI
```

## Demo Updates

- **Act 3 (Client):** Show SOC workflow funnel and SOC Assistant sidebar

## Known Limitations

- Client SOC Assistant uses mock AI when `OPENAI_API_KEY` is unset
- PDF export still uses browser print (no server-side PDF)
- Seed script does not run on existing databases
- Chat sidebar stacks below dashboard on screens narrower than `xl`
