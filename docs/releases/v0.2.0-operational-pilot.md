# TrustOps v0.2.0-operational-pilot Release Notes

**Release date:** June 2026  
**Codename:** Operational Pilot

## Overview

TrustOps v0.2.0-operational-pilot makes the platform **usable and credible during a real 30-day MDR/SOCaaS pilot**. v0.1 proved the demo; v0.2 proves **operational usefulness** — visibility, auditability, case quality, trust-metric drilldown, and buyer-facing polish.

## What Changed

Building on v0.2.0-pilot client portal features (workflow funnel, SOC Assistant, integration hardening), this release adds manager-facing operational tooling.

### Operational Visibility

- **Integration Health Dashboard** (`/app/integrations`, `GET /integrations/status`)
- Status cards for Microsoft Sentinel and Generic Webhook
- Alerts/failures in last 24 hours, API key and client mapping indicators

### Auditability

- **Audit Log Viewer** (`/app/audit`, `GET /audit-logs`)
- Filter by client, case, event type, date range, search
- Expandable previous/new value JSON
- SOC Manager and Platform Admin only

### Case Quality

- **Case Quality Score** (0–100) with transparent formula
- Quality flags: missing evidence, no decision, SLA breach, human-AI disagreement, etc.
- Visible in Case Queue, Case Detail, Manager Dashboard
- **Not exposed to client users**

### Trust Metrics Drilldown

- `GET /dashboards/trust-metrics/drilldown`
- Clickable metric cards on Trust Metrics page
- Open underlying cases for high-conf rejections, disagreements, overrides, QA reversals
